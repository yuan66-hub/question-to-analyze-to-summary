# 知识库质量治理策略

## 背景

知识库内容错误或过期会对AI产生严重误导：
- **幻觉强化** - AI把错误信息当作事实依据进行推理
- **错误传播** - 基于错误知识生成的答案进一步污染对话
- **用户信任损耗** - 错误答案导致用户对系统失去信任

核心原则：**"不可靠输入 → 不可靠输出"** - 知识库质量是AI答案质量的上限

---

## 一、内容层治理

### 1.1 版本标记体系
每个文档应标注：
| 字段 | 说明 |
|------|------|
| `created_at` | 创建时间 |
| `updated_at` | 最后更新时间 |
| `expires_at` | 过期时间（可选） |
| `source` | 来源/作者 |
| `confidence_level` | 可信度等级（高/中/低） |

### 1.2 生命周期管理
```
内容创建 → 定期审核 → 过期标记 → 下架/更新
    ↑
    └── 触发修正流程（用户反馈/主动审计）
```

- 设置内容过期时间，自动触发审核提醒
- 长期未更新的内容自动降权

### 1.3 来源分级
| 等级 | 来源类型 | 权重 |
|------|----------|------|
| L1 | 官方文档、权威机构发布 | 1.0 |
| L2 | 经过验证的专业资料 | 0.8 |
| L3 | 用户贡献、公开社区内容 | 0.5 |

---

## 二、检索层治理

### 2.1 时效性过滤
- 检索时按 `updated_at` 降序排列
- 设置时间窗口（如近1年），过期内容自动降权
- 高频访问但未更新的内容触发告警

### 2.2 置信度评分
检索结果综合评分：

```
score = relevance × confidence_level × freshness × source_weight
```

其中：
- `relevance` - 语义相似度
- `confidence_level` - 内容可信度等级
- `freshness` - 时效性因子（随时间衰减）
- `source_weight` - 来源权重

### 2.3 冲突检测
- 多个检索结果内容矛盾时，主动标记而非随意选择
- 策略：优先高可信度来源，矛盾时返回多个选项让用户判断

---

## 三、生成层治理

### 3.1 溯源显式化
生成答案时附带参考来源：
```
答案: xxx

参考来源:
- [文档A](url) - 可信度：高 | 更新时间：2024-01
- [文档B](url) - 可信度：中 | 更新时间：2023-06
```

### 3.2 不确定性表达
对低可信度内容明确标注：
- "该信息来自用户贡献，未经专业验证，请谨慎参考"
- "此内容最后更新于2023年，可能已过时"

### 3.3 置信度阈值
- 设置最低置信度阈值（如 0.3）
- 低于阈值的结果不参与生成
- 低于阈值时主动回复："当前知识库中无足够可信的信息来回答此问题"

---

## 四、运营层治理

### 4.1 定期审计机制
| 审计类型 | 频率 | 内容 |
|----------|------|------|
| 主动抽检 | 每月 | 随机抽查知识库内容准确性 |
| 全量审核 | 每季度 | 全面检查内容时效性 |
| 专项审计 | 按需 | 重大版本更新后验证相关文档 |

### 4.2 用户反馈闭环
```
用户标记错误 → 反馈入库 → 优先级评估 → 修正/下架 → 结果反馈
```

- 提供便捷的"内容纠错"入口
- 对有效反馈的用户给予积分/奖励
- 修正结果通过消息通知反馈用户

### 4.3 A/B验证
- 保留正确知识库版本作为对照组
- 新版知识库上线前进行对比测试
- 指标：错误率下降比例、用户满意度

---

## 五、治理效果评估

### 核心指标
| 指标 | 计算方式 | 目标 |
|------|----------|------|
| 内容错误率 | 错误内容数 / 总内容数 | < 1% |
| 答案准确率 | 用户正面反馈 / 总反馈 | > 95% |
| 过期内容占比 | 超过X年未更新内容 / 总内容 | < 5% |
| 反馈响应率 | 已处理反馈 / 总反馈 | > 90% |

### 监控告警
- 错误内容被使用超过N次时触发告警
- 单一来源占比过高时预警（避免单点依赖）
- 用户投诉率突然上升时自动冻结相关知识

---

## 六、实施优先级

### Phase 1 - 基础治理（1-2周）
- [ ] 建立内容元数据体系（时间戳、来源、版本）
- [ ] 接入用户反馈入口
- [ ] 实现基础的置信度评分

### Phase 2 - 中级治理（1个月）
- [ ] 时效性自动过滤
- [ ] 溯源显式化输出
- [ ] 冲突检测机制

### Phase 3 - 高级治理（持续优化）
- [ ] A/B验证体系
- [ ] 定期审计自动化
- [ ] 智能内容推荐更新

---

## 总结

知识库质量治理是系统性工程，核心要点：

1. **内容为本** - 建立完善的版本标记与生命周期管理
2. **检索过滤** - 时效性与可信度双重评分
3. **生成可控** - 溯源透明、不确定时主动告知
4. **运营闭环** - 用户反馈驱动持续优化
